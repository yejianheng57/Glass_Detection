# 玻璃空洞实例分割项目方案

## 1. 项目目标

本项目用于玻璃上下料场景中的视觉检测。检测对象是一块放置在桌面上的玻璃圆盘，玻璃外圈带齿，中间存在多个空洞。空洞形状不一定是标准长方形，可能存在不规则边界、透视变形、镜头畸变或加工误差。相机可以固定在桌面上方俯拍玻璃，也可以安装在机械臂末端，通过机械臂在不同位姿下从不同视角观察玻璃并采集 RGB 图像。

最终目标不是检测空洞的矩形框或固定 4 个角点，而是对每个空洞进行实例分割，得到每个空洞在图像中的多边形轮廓区域。

每个空洞定义为一个实例目标。一个实例由人工沿空洞边界点击的多个轮廓点组成：

```text
点 1 -> 点 2 -> 点 3 -> ... -> 点 N -> 结束当前实例
```

标注点应按顺时针或逆时针顺序沿同一个空洞边界连续点击。每个实例至少需要 3 个点，点数不固定，应根据空洞边界复杂程度决定。

## 2. 硬件与采集条件

相机型号：Intel RealSense D435i。

采集数据时使用 D435i 的 RGB 图像作为模型输入。深度图暂不作为第一阶段训练输入，后续如果需要进行空间坐标换算、玻璃平面定位或机械臂抓取定位，可以再结合深度数据、相机标定参数、机械臂位姿和手眼标定结果。

采集方式支持两种：

- 固定相机采集：相机安装在桌面上方，相机位置和姿态保持不变。
- 机械臂末端相机采集：相机安装在机械臂末端，机械臂移动到不同位姿后采集图像。

采集时需要尽量保证：

- 固定相机方案中，相机尽量固定不动，并尽量垂直于桌面。
- 机械臂末端相机方案中，需要记录每张图像对应的机械臂位姿。
- 桌面和玻璃保持平整。
- 光照稳定，尽量减少玻璃反光。
- 玻璃在桌面上的不同位置、角度都要采集。
- 如果使用机械臂末端相机，还需要采集不同观察高度、观察角度和相机到玻璃距离下的数据。
- 图像中空洞边界应尽量清晰可见。

即使相机和桌面尽量保持水平，实际图像中仍可能存在透视变形和镜头畸变。如果相机安装在机械臂末端，不同机械臂位姿会带来更明显的视角变化和透视变化。因此本项目不假设空洞在图像中一定是标准矩形，而是通过实例分割轮廓表达其真实图像区域。

## 3. 总体技术路线

项目流程如下：

```text
选择固定相机或机械臂末端相机采集方案
采集 RGB 图像，并在需要时记录机械臂位姿
人工标注每个空洞的多边形轮廓
整理 YOLO Seg 数据集
使用 YOLO26 Seg 训练实例分割模型
验证模型在新图像上的检测效果
部署模型进行实时检测
```

模型方案：

- 模型类型：YOLO Seg 实例分割。
- 模型版本：YOLO26。
- 检测类别：1 类，建议命名为 `glass_hole`。
- 每个目标标注形式：一个多边形轮廓，点数不固定。
- 模型输出：每个空洞的目标框、置信度和实例分割掩膜。

YOLO26 不是直接拿来就能识别本项目中的玻璃空洞。它需要使用本项目采集并标注的数据进行训练或微调。官方预训练模型只提供通用视觉特征能力，真正识别玻璃空洞实例轮廓需要依赖自有数据集。

## 4. 数据采集工具

数据采集工具自定义编写，建议使用 Python + OpenCV + RealSense SDK。

主要功能：

- 连接 Intel RealSense D435i。
- 打开 RGB 实时预览画面。
- 按 `Enter` 保存当前 RGB 图像。
- 自动生成图片文件名，避免覆盖。
- 支持按 `Esc` 或 `q` 退出。
- 保存采集日志，记录图片编号、采集时间、相机参数和采集模式。
- 如果相机安装在机械臂末端，需要同步记录每张图片对应的机械臂末端位姿。

建议保存路径：

```text
datasets/glass_rect/raw_images/
```

建议图片命名：

```text
glass_000001.jpg
glass_000002.jpg
glass_000003.jpg
```

第一阶段建议先采集 200-500 张图像，用于验证完整流程。如果效果不稳定，再针对失败场景补充采集数据。

如果采用机械臂末端相机采集，建议在采集日志中至少保存：

```text
image_name
timestamp
camera_serial
robot_pose_x
robot_pose_y
robot_pose_z
robot_pose_rx
robot_pose_ry
robot_pose_rz
```

如果暂时只训练图像实例分割模型，机械臂位姿不会直接进入 YOLO26 Seg 训练，但它对后续坐标转换、误差分析和机械臂闭环定位非常重要。

## 5. 数据标注工具

数据标注工具自定义编写，不使用 LabelImg。LabelImg 主要适合矩形框检测，不适合本项目的实例分割多边形标注。

标注工具建议使用 Python + OpenCV 或 Python + PyQt 编写。

主要功能：

- 加载采集好的图片。
- 鼠标沿每个空洞边界连续点击多个轮廓点。
- 按指定按键确认当前实例已经点完，然后开始下一个实例。
- 在图像上实时显示点、连线、多边形和目标编号。
- 支持撤销上一个点。
- 支持删除当前目标。
- 支持上一张、下一张图片切换。
- 支持保存 YOLO Seg 格式标签。
- 支持重新打开已有标签继续修改。

推荐标注方式：

```text
沿空洞边界顺时针或逆时针依次点击多个点
按 f 或 Enter 结束当前空洞实例
继续点击下一个空洞实例
```

保存格式采用 YOLO Seg 格式。每张图片对应一个同名 `.txt` 标签文件。

示例：

```text
glass_000001.jpg
glass_000001.txt
```

YOLO Seg 单行标签格式：

```text
class_id x1 y1 x2 y2 x3 y3 ... xN yN
```

说明：

- `class_id`：类别编号，本项目只有 1 类，因此为 `0`。
- `x1 y1 ... xN yN`：当前空洞实例的多边形轮廓点坐标，全部归一化到 0-1。
- 每个实例至少 3 个点，点数不固定，越复杂的边界可以标更多点。

## 6. 数据集结构

建议最终整理为 YOLO Seg 训练目录：

```text
datasets/
  glass_rect/
    images/
      train/
      val/
    labels/
      train/
      val/
    data.yaml
```

`data.yaml` 内容示例：

```yaml
path: datasets/glass_rect
train: images/train
val: images/val

names:
  0: glass_hole
```

YOLO Seg 不需要配置 `kpt_shape`。每个标签行直接使用多边形点表示一个实例。

## 7. 模型训练

使用 Ultralytics YOLO26 Seg 进行训练。

安装依赖：

```bash
pip install ultralytics
```

训练命令示例：

```bash
yolo segment train data=datasets/glass_rect/data.yaml model=yolo26n-seg.pt epochs=100 imgsz=640
```

如果显卡性能较弱，可以优先使用 `yolo26n-seg.pt`。如果数据量更大、算力更强，可以尝试 `yolo26s-seg.pt` 或更大的模型。

训练完成后，需要使用验证集和实际新采集图像检查：

- 是否漏检某些空洞实例。
- 是否误检桌面、反光、玻璃边缘。
- 分割掩膜是否稳定贴合真实空洞边界。
- 不同位置、不同旋转角度下是否稳定。
- 玻璃靠近画面边缘时是否仍然准确。

## 8. 后续部署

部署阶段使用 D435i 实时采集 RGB 图像，将图像输入训练好的 YOLO26 Seg 模型，输出每个空洞实例的目标框、置信度和分割掩膜。

后续如果需要将图像坐标转换为桌面物理坐标，固定相机方案需要增加：

- 相机内参标定。
- 镜头畸变校正。
- 相机到桌面平面的单应性变换。
- 像素坐标到实际毫米坐标的转换。

如果使用机械臂末端相机方案，还需要增加：

- 相机内参标定。
- 镜头畸变校正。
- 机械臂末端到相机的手眼标定。
- 机械臂基坐标系、相机坐标系、桌面坐标系之间的坐标变换。
- 每张图像对应的机械臂位姿同步。

如果后续要服务于机械臂或上下料设备，建议在模型输出分割掩膜后，再增加几何后处理，例如计算每个空洞的轮廓、中心点、主方向、面积和旋转角度。

## 9. 当前结论

本项目最终采用以下方案：

- 自定义数据采集工具。
- 自定义实例分割数据标注工具。
- 使用 Intel RealSense D435i 采集 RGB 图像。
- 相机可以固定俯拍，也可以安装在机械臂末端从不同位姿观察玻璃。
- 每个空洞标注一个多边形轮廓，点数不固定。
- 使用 YOLO26 Seg 训练实例分割模型。
- 通过模型输出的分割掩膜表达可能不规则或畸变后的空洞区域。
